Oracle数据库基础之一

发表于 2017-04-17 | 分类于 Oracle

数据库原理

数据库简介

文件存储方式保存数据的弊端

- 缺乏对数据的整体管理
- 不利于数据分析和共享
- 大量数据不能长期保存在文件中

DB和DBMS

- 数据库(DB)：按照数据结构来组织、存储和管理数据的仓库
- 数据库管理系统(DBMS)：管理数据库的软件

关系数据库简介

- 关系：描述两个元素间的关联或对应关系，使用关系模型把数据组织到二维数据表中
- 产品：MySQL/DB2/MSSQL/Oracle...

表的概念

- 一个关系数据库由多个数据表组成，是关系数据库的基本存储结构
- 表由行和列组成，行被称为记录(Record)，列被称为字段(Field)
- 表和表之间存在关联关系

主流关系型数据库

- Oracle概述
    + Oracle公司开发的关系型数据库产品
    + 世界上第一个商品化的关系型数据库管理系统
    + 支持多数据类型，提供面对对象的数据支持，支持多平台
- DB2概述
    + IDM公司开发的关系型数据库管理系统
    + 支持多平台、多数据类型
- Sybase概述
    + 美国Sybase公司开发的关系型数据库系统
    + 支持多平台，C/S结构
- MSSQL概述
    + 微软公司开发的产品
    + 只能在windows上运行
- Mysql概述
    + 开源的小型关系型数据库管理系统

SQL概述

结构化查询语言

- Structured Query Language，简称SQL
- SQL是在关系型数据库上执行各种操作的标准语言
- 所有数据库都使用相同或者相似的语言
- SQL
    - 数据定义语言(DDL):Data Definition Language
    - 数据操作语言(DML):Data Manipulation Language
    - 事务控制语言(TCL):Transaction Control Language
    - 数据查询语言(DQL):Data Query Language
    - 数据控制语言(DCL):Data Control Language

数据定义语言(DDL)

- 用于建立、修改、删除数据库对象
- CREATE：创建表或其他对象的结构
- ALTER：修改表或其他对象的结构
- DROP：删除表或其他对象的结构
- TRUNCATE：删除表数据，保留表结构

数据操作语言(DML)

- 用于改变数据表中的数据，和事务相关，执行完后需要经过事务控制语句提交后才能将修改应用到数据库中
- INSERT：将数据插入到数据表中
- UPDATE：更新数据表中已存在的数据
- DELETE：删除数据表中的数据

事务控制语言(TCL)

- 用来维护数据一致性的语句
- COMMIT：提交
- ROLLBACK：回滚
- SAVEPOINT：保存点，使当前的事务可以回退到指定的保存点，便于取消部分改变

数据查询语言(DQL)

- 用来查询所需要的数据
- SELECT：查询

数据控制语言(DCL)

- 用来执行权限的兽欲和收回操作
- GRANT：用于给用户或角色授权
- REMOVE：用于收回用户或角色已有的权限
- CREATE USER：创建用户
